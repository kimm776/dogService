controller

목적:
컨트롤러 계층의 테스트를 작성하는 폴더입니다.
컨트롤러는 클라이언트의 요청을 처리하는 역할을 하므로, 이 폴더에서는 주로 웹 요청/응답을 테스트하게 됩니다.

테스트 종류:
1. 통합 테스트(Integration Test)
스프링 컨텍스트를 로드하여 컨트롤러와 관련된 모든 레이어(서비스, 리포지토리 등)를 통합적으로 테스트합니다.
주로 @SpringBootTest와 @AutoConfigureMockMvc 어노테이션을 사용합니다.

2. 단위 테스트(Unit Test)
컨트롤러 자체의 로직만을 테스트하기 위해 서비스 계층을 목(Mock)으로 대체하여 테스트합니다.
 @WebMvcTest 어노테이션을 사용해 스프링 MVC 구성 요소만 로드합니다.
